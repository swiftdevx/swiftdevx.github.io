ARKit 初体验

发表于 2018-03-05 | 分类于 iOS ， ARKit

Augmented Reality

增强现实创造了一种虚拟物体被放置在一个物理世界中的假象。
Augmented reality is creating the illusion that virtual objects are placed in a physical world.

ARKit

  • 追踪 / Tracking
  • 场景理解 / Scene Understanding
  • 渲染 / Rendering

基础

ARKit 是一个基于会话的 API ，所以 ARSession 控制着所有与 AR 有关的处理，而 ARSessionConfiguration 就是配置会话的类。在 ARSession 底层的是 AVCaptureSession 和 CMMotionManager ，分别用来捕捉图像数据和动作数据为追踪提供支持。一旦处理完成，ARSession 就会输出 ARFrames 。ARFrames 是一个快照，包含了会话的所有状态和所有需要渲染的东西。

ARSessionConfiguration

ARSessionConfiguration 决定了运行在会话上的追踪种类，它的基类提供了三种程度的自由追踪，其实就是设备的方向。它的子类 ARWorldTrackingSessionConfiguration 提供了六种程度的自由追踪，包括了设备的方向和相对位置。得到这些信息后就能实现场景理解，例如特征点和物理位置。
可以设置会话配置类的属性来启用或禁用特性。
还可以通过可用性检查设备是否支持世界追踪。

ARSession

ARSession 管理所有的 AR 处理，除了使用 configuration 作为参数调用 run 以外，还可以调用 pause ，再次调用 run 继续会话。为了更换不同的配置，可以多次调用 run 。这里还提供了 run 选项重置追踪。

通过设置代理，你可以执行 session 在不同状态下的方法，进而做相应的处理。

除了执行 didUpdate 方法获取当前的 ARFrame ，还可以直接访问 session 的 currentFrame 属性。

ARFrame

每一个 ARFrame 对象都包含了所有需要渲染的东西。

  1. 摄像头图像：用来渲染场景的背景
  2. 追踪信息：设备方向、位置，甚至追踪状态
  3. 场景理解：特征点、空间中的物理位置、光线模拟等信息
    空间中的物理位置就是用 ARAnchor 表示的。

    ARAnchor

    ARAnchor 是一个在空间中相对的或实际的位置和方向，可以在场景中添加或移除。它通常表示一个被固定在物理环境中的虚拟内容。

    Tracking

    追踪功能是实时在空间中确定一个物理方位，这个方位是你的设备的位置和方向，它关系到渲染的效果。ARKit 的这个技术用的是视觉惯性测深法，持续更新投影矩阵来渲染虚拟内容。
    更加厉害的是，它提供了现实世界的比例。虚拟物体可以更好地融入现实中，你的设备移动的距离是用米作单位。
    还有所有通过追踪得到的位置都是相对于会话的开始位置。
    追踪的工作原理是通过 3D 特征点来实现。AR 在运行过程中，能够检测场景中的特征点，通过移动场景来对它们进行三角测量，设备与匹配的特征点间会存在（看不见的）连线。利用所有的这些信息加上动作数据，你可以得到设备精确的方位。

    Scene Understanding

    场景理解是寻找更多与环境相关的信息以便在这个环境中放置虚拟对象。这些信息包括环境 3D 拓扑，光照条件等。
  • 平面检测：包括水平面和垂直面的检测，随着设备的移动会检测更多的平面，直到覆盖整个水平面的边缘。当然也可以选择扩展平面为无限平面。检测出一个新平面后，ARPlaneAnchor 会被放置在表面上，它是 ARAnchor 的子类，代表真实世界的位置和方向。

  • 碰撞测试：设备发送一条射线与现实世界相交形成交叉点，利用所有的这些场景信息，还包括平面检测和 3D 特征点去找出虚拟内容的位置。代码实现是用归一化的图像空间坐标中的点表示射线，即坐标范围从(0, 0)到(1, 1)之间。

  • 光照估计：使用曝光信息来确定图像的相对亮度。通过设置 ARSessionConfiguration 的 isLightEstimationEnabled 属性来开关此功能。光照估计的结果会放在 ARFrame 的 lightEstimate 属性中。

    Rendering

    渲染将追踪和场景理解与内容结合在一起。为了使用 ARKit 进行渲染，需要处理 ARFrame 中提供的所有信息。
    如果使用 SceneKit 和 SpriteKit ，ARKit 已经创建了自定义视图，负责呈现 ARFrame。
    以 SceneKit 为例，它有 ARSCNView 这个视图类，是 SCNView 的子类，包含一个负责更新渲染的 ARSession 。至于 ARSCNView 中的摄像头、光照、锚点等都是与 SCNView 的一一对应。
